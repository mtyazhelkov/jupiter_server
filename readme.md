Запуск jupiter notebook через docker-compose. Пароль зашифрован в .env файле.<br>
В docker-compose.yml файле в deploy заданы ограничения RAM и CPU, чтобы сервер не падал на перегруженных скриптах.<br>
Эти ограничения можно пересмотреть в большую/меньшую сторону в дальнейшем.<br>
**Внимание!** Перед запуском  необходимо выдать права на папку с ноутбуками: sudo chmod -R 777 notebooks.<br>
Почему так происходит?<br>
Внутри Docker-контейнера Jupyter работает под пользователем jovyan (UID 1000), а на  сервере папка принадлежит root (UID 0). Из-за этого возникает конфликт. 777 убирает все преграды: контейнер видит, что запись разрешена «всем», и спокойно сохраняет файлы.
